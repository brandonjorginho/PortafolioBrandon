Duodécima semana

Desarrollada del, 25 de Diciembre, al 29 de Diciembre

¿Que temas aprendió?

Durante el desarrollo de esta semana, se aprendió a utilizar useReducer y useContext como herramientas clave. Se realizaron los siguientes procedimientos: se implementó useReducer para manejar el estado de la aplicación, creando un reducer para gestionar acciones como agregar usuarios; se modularizó la lógica de manejo de formularios con el custom hook useForm; se estructuraron los componentes en archivos separados para mejorar la organización y reutilización del código; se introdujo el uso de referencias (useRef) para acceder directamente a nodos del DOM; se consumieron APIs externas con la biblioteca Axios; se emplearon alertas visuales con la librería SweetAlert2. Además, se utilizó useContext para pasar valores de estado de manera eficiente entre componentes, separando así la lógica de presentación y facilitando la estructuración del proyecto. En resumen, se logró un proyecto bien organizado, donde se aplicaron conceptos fundamentales de React para desarrollar un CRUD funcional y modular.

Definiciones

useReducer

Hook de React que permite manejar el estado de la aplicación de una manera más compleja que useState, útil para casos donde el estado depende del anterior o se deben realizar acciones más complejas.

useContext

Otro Hook de React que proporciona una forma de pasar datos a través del árbol de componentes sin necesidad de pasar props manualmente en cada nivel. Reducer: Una función que especifica cómo el estado de una aplicación cambia en respuesta a acciones enviadas a través de dispatch en useReducer.

Custom Hook

Funciones personalizadas en React que encapsulan lógica específica y pueden ser reutilizadas en varios componentes.

useRef

Hook que proporciona una forma de acceder a nodos del DOM directamente en lugar de depender del estado de React, útil para realizar operaciones directas sobre elementos HTML.

Procedimientos

  1. Implementación de useReducer: Se integró useReducer para gestionar el estado de la aplicación, permitiendo transiciones de estado más complejas y manejando acciones específicas.
  2. Creación de useForm: Se desarrolló el custom hook useForm para modularizar la lógica de manejo de formularios, facilitando la captura y actualización de valores de formulario.
  3. Estructuración de componentes: Se organizó la aplicación con componentes en archivos separados para mejorar la reutilización y mantenimiento del código, siguiendo buenas prácticas de estructuración.
  4. Consumo de APIs con Axios: Se utilizó la biblioteca Axios para realizar peticiones HTTP, permitiendo la conexión y obtención de datos desde APIs externas.
  5. Uso de referencias (useRef): Se emplearon referencias para acceder directamente a nodos del DOM, evitando la necesidad de depender del estado de React para obtener y manipular elementos específicos.

Resultados

  1. Manejo eficiente del estado: A través de la implementación de useReducer en lugar de useState, se logró un manejo más estructurado y escalable del estado en la aplicación, permitiendo una transición suave entre asignar estados y despachar acciones.
  2. Modularización de la lógica de formularios: La creación y aplicación del custom hook useForm facilitó la modularización y reutilización de la lógica asociada a los formularios. Esto simplificó el manejo de cambios en los inputs y el reinicio de los valores del formulario.
  3. Consumo eficaz de APIs externas: Se utilizó la biblioteca Axios para realizar solicitudes a APIs externas, proporcionando un manejo robusto de las operaciones asíncronas y mejorando la integración de datos externos en la aplicación React.
  4. Acceso directo a nodos del DOM: La incorporación de useRef permitió acceder directamente a los nodos del DOM, eliminando la necesidad de depender del estado de React para operaciones específicas. Esto mejoró la eficiencia en la interacción con elementos HTML sin desencadenar re-renderizaciones innecesarias.
  5. Estructuración clara de componentes: La organización de los componentes en archivos separados facilitó la reutilización y el mantenimiento del código. La separación de responsabilidades en componentes específicos mejoró la legibilidad y escalabilidad del proyecto.
  6. Implementación de contextos y reducers: Se aplicó el uso de contextos y reducers para compartir y manejar el estado de manera global en la aplicación, facilitando la comunicación entre componentes y permitiendo una gestión más eficiente de las actualizaciones de estado.
