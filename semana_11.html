Undécima semana

Desarrollada del, 18 de Diciembre, al 22 de Diciembre

¿Que temas aprendió?

Durante esta semana, se exploraron conceptos clave en React, incluyendo el uso efectivo del hook useEffect para gestionar efectos secundarios. Se aprendió a modularizar la lógica de formularios mediante custom hooks, como useForm, y se adquirió la habilidad de consumir APIs externas con Axios. La utilización de referencias (useRef) permitió acceder directamente a nodos del DOM. Se implementaron contadores y generadores de números aleatorios para interactuar con APIs, y se promovió la estructuración de componentes en archivos separados para mejorar la reutilización y el mantenimiento del código.

Definiciones

Hook useEffect:

Un hook de React que permite ejecutar efectos secundarios en componentes funcionales, como realizar solicitudes HTTP, manipular el DOM o suscribirse a eventos, en diferentes puntos del ciclo de vida del componente.

Custom Hook

Una función personalizada en React que puede contener lógica de estado y efectos, permitiendo encapsular y reutilizar comportamientos específicos en múltiples componentes.

Referencia (useRef)

Un objeto proporcionado por React que permite mantener una referencia mutable a un elemento del DOM. Se utiliza para acceder y modificar directamente propiedades o estilos del elemento sin afectar el estado de React.

Axios

Una biblioteca JavaScript utilizada para realizar solicitudes HTTP en aplicaciones basadas en el navegador y en Node.js. Es comúnmente empleada para consumir APIs externas.

Procedimientos

  1. Implementación de efectos con useEffect para distintos escenarios, incluyendo casos de renderización, cambios en variables específicas y ejecución única al cargar el componente.
  2. Creación de un custom hook llamado useForm para gestionar formularios de manera más eficiente y reutilizable.
  3. Consumo de la API de Breaking Bad para mostrar cotizaciones al recargar el componente y actualización de la interfaz mediante un botón de recarga.
  4. Creación de un proyecto que consume la API de Pokémon, mostrando información de un Pokémon específico según un número aleatorio o seleccionado por el usuario.
  5. Utilización de Axios para realizar solicitudes HTTP a la API de Pokémon y obtener datos dinámicos.
  6. Manipulación del DOM mediante el uso de referencias (useRef) para cambiar estilos o interactuar con elementos específicos.

Resultados

  1. Se logró crear un proyecto que consume la API de Breaking Bad y otra que interactúa con la API de Pokémon, demostrando la capacidad para consumir datos externos en aplicaciones React.
  2. Se implementaron contadores y generadores de números aleatorios para enriquecer la interactividad de las aplicaciones y proporcionar funcionalidades más dinámicas.
  3. Se mejoró la estructura del código al dividir componentes en archivos separados y aplicar buenas prácticas de desarrollo.
  4. Se reforzaron conceptos clave de React, como el uso de useState, useEffect, custom hooks y la manipulación eficiente del DOM.
