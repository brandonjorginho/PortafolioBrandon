Decimocuarta semana

Desarrollada del, 08 de Enero, al 12 de Enero

¿Que temas aprendió?

En esta semana, se abordó la implementación de autenticación con Next.js utilizando NextAuth. Se instaló y configuró el backend del sistema, se exploró la documentación de NextAuth para proveedores de credenciales y se procedió a crear el frontend con Bootstrap. La configuración de NextAuth incluyó la creación de proveedores de credenciales para autenticación mediante correo electrónico y contraseña. Se instaló y utilizó el paquete Bootstrap para mejorar el diseño y la apariencia del frontend. También se implementó la gestión de sesiones y se exploraron conceptos como el proveedor de sesiones y el middleware de NextAuth.

Definiciones

NextAuth

Librería utilizada con Next.js para la implementación de autenticación y autorización en aplicaciones web.

Backend

Parte de una aplicación que se encarga de procesar datos y realizar operaciones del lado del servidor.

Frontend

Interfaz de usuario y experiencia de usuario de una aplicación, visible y accesible para el usuario final.

Bootstrap

Marco de diseño que facilita el desarrollo de interfaces web responsivas y atractivas.

Middleware

Software que se encuentra entre el sistema operativo y las aplicaciones, facilitando la comunicación y el procesamiento de datos.

Proveedor de sesiones

Componente que gestiona la información de la sesión del usuario, como su estado de autenticación.

Procedimientos

  1. Instalación y configuración del backend del sistema, incluyendo la ejecución de comandos como npm i y docker compose up -d.
  2. Creación de un proveedor de sesiones para gestionar la autenticación y autorización del usuario.
  3. Configuración de proveedores de credenciales para autenticación mediante correo electrónico y contraseña.
  4. Instalación del paquete Bootstrap para mejorar el diseño y la apariencia del frontend.
  5. Implementación de un componente de botones para manejar la autenticación y desautenticación del usuario.
  6. Configuración de middleware para gestionar rutas protegidas en el dashboard de la aplicación.

Resultados

  1. Se logró instalar y configurar el backend del sistema, así como la creación del frontend utilizando Bootstrap para mejorar la interfaz de usuario.
  2. NextAuth se implementó exitosamente para gestionar la autenticación mediante correo electrónico y contraseña.
  3. Se diseñó y creó una página de dashboard protegida, la cual solo es accesible para usuarios autenticados.
  4. Se implementaron botones de autenticación y desautenticación, permitiendo al usuario interactuar con el sistema de autenticación.
  5. Se exploraron y aplicaron conceptos importantes, como el proveedor de sesiones y el middleware, para gestionar el estado de autenticación y proteger rutas específicas en la aplicación.
