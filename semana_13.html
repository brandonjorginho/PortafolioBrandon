Decimotercera semana

Desarrollada del, 01 de Enero, al 05 de Enero

¿Que temas aprendió?

En esta semana, se abordaron diversos temas relacionados con Next.js, un marco de React para aplicaciones web de pila completa. Se exploraron las Single Page Applications (SPAs) y su desventaja en términos de SEO, destacando cómo Next.js aborda este problema. Además, se enseñó cómo utilizar Server Components en Next.js para consumir la API de Rick and Morty, descargando proyectos desde GitHub, instalando dependencias y realizando solicitudes HTTP con fetch. Se introdujo el concepto de Fetch en Server Components y se detalló la creación de funciones para obtener datos de la API y renderizarlos en componentes.

Definiciones

Next.js

Un marco de React para crear aplicaciones web de pila completa, que combina la creación de interfaces de usuario mediante React Components con funciones y optimizaciones adicionales proporcionadas por Next.js.

Single Page Applications (SPAs)

Aplicaciones web que cargan una sola página y actualizan su contenido dinámicamente a medida que los usuarios interactúan con la aplicación, sin necesidad de recargar la página completa.

SEO-friendly

Característica que indica que una aplicación web o sitio es fácilmente indexable por los motores de búsqueda, permitiendo una mejor visibilidad en los resultados de búsqueda.

API REST

Interfaz de programación de aplicaciones basada en el estilo arquitectónico Representational State Transfer (REST), que utiliza métodos HTTP para realizar operaciones en recursos.

Incremental Static Regeneration (ISR)

Interfaz de programación de aplicaciones basada en el estilo arquitectónico Representational State Transfer (REST), que utiliza métodos HTTP para realizar operaciones en recursos.

Server Components

Componentes en Next.js que se ejecutan en el servidor y permiten realizar operaciones del lado del servidor, como el acceso a bases de datos o la obtención de datos externos.

Procedimientos

  1. Descarga de proyectos desde GitHub y configuración del entorno.
  2. Instalación de dependencias necesarias para el proyecto, incluyendo el uso de comandos como npm install.
  3. Creación de funciones para realizar solicitudes a una API externa utilizando fetch.
  4. Utilización de Server Components en Next.js para obtener y renderizar datos de la API de Rick and Morty.
  5. Uso de Next Image para mostrar imágenes de personajes con propiedades obligatorias como src, width, height, y alt.
  6. Implementación de vinculación de enlaces para navegar entre páginas de personajes.

Resultados

  1. Se logró consumir la API de Rick and Morty desde un Server Component en Next.js, obteniendo datos de personajes y mostrándolos en la interfaz.
  2. Se implementó la generación estática de páginas utilizando ISR, lo que permitió mejorar la velocidad de carga al conservar beneficios de generación estática y actualizar páginas de manera incremental.
  3. Se creó una página detallada para cada personaje, incluyendo información como nombre, especie y una imagen.
  4. Se aplicó la técnica de vinculación de enlaces para facilitar la navegación entre las páginas de personajes.
